# Changelog

## 0.5.0

### Добавлено

- Добавлено сохранение состояния конвертера между запусками приложения.
- Добавлено восстановление выбранного режима после перезапуска.
- Добавлено восстановление направления конвертации после перезапуска.
- Добавлено восстановление выбранного алфавита после перезапуска.
- Добавлено восстановление смещения после перезапуска.
- Добавлено сохранение истории операций между запусками приложения.

### Улучшено

- История операций больше не теряется после закрытия приложения.
- Очистка истории теперь сохраняется между запусками.
- Последние настройки пользователя восстанавливаются автоматически при следующем открытии приложения.
- Сохранены существующие правила истории: максимум 20 операций, игнорирование пустого ввода/результата и защита от подряд идущих дублей.
- Исправлены триггеры GitHub Actions: обычные push в `develop` и `main` больше не создают лишние CI-запуски.

### Техническое

- Добавлен слой хранения состояния приложения.
- Добавлено persistent-хранилище для настроек конвертера.
- Добавлено persistent-хранилище для истории операций.
- Чистая логика истории сохранена отдельно от UI.
- Core-конвертеры не зависят от Android-хранилища.
- CI workflow теперь запускается вручную или на pull request.
- Release workflow запускается по тегам `v*`.

### Проверено

- Локальная сборка проходит через `assembleDebug`.
- Unit-тесты проходят через `testDebugUnitTest`.
- Приложение запускается на Android Emulator.
- Проверено восстановление настроек после перезапуска.
- Проверено сохранение истории после перезапуска.
- Проверено удаление истории после перезапуска.
- Основной конвертер, справка и история продолжают работать.

## 0.4.1

### Исправлено

- Исправлены триггеры GitHub Actions.
- Обычный `push` в ветку `develop` больше не запускает CI-сборку.
- Release workflow теперь запускается только при публикации тегов `v*` или вручную, если включён `workflow_dispatch`.

### Техническое

- Уточнены события запуска `.github/workflows/android-ci.yml`.
- Уточнены события запуска `.github/workflows/android-release.yml`.
- Снижено количество лишних запусков GitHub Actions во время обычной разработки.

### Проверено

- CI workflow доступен для ручного запуска.
- Release workflow не запускается на обычный push в `develop`.
- Локальная сборка проходит через `assembleDebug`.
- Unit-тесты проходят через `testDebugUnitTest`.

## 0.4.0

### Добавлено

- Добавлена автоматическая CI-сборка Android-проекта через GitHub Actions.
- Добавлен запуск unit-тестов в GitHub Actions.
- Добавлена автоматическая сборка debug APK в GitHub Actions.
- Добавлена публикация debug APK в GitHub Releases при создании тега `v*`.
- Добавлена возможность вручную запускать CI workflow через `workflow_dispatch`.

### Улучшено

- APK теперь можно скачать из GitHub Actions для обычных сборок.
- APK релизных версий теперь можно скачать из GitHub Releases.
- Добавлена документация по процессу автоматической сборки и релизов.

### Техническое

- Добавлен workflow `.github/workflows/android-ci.yml`.
- Добавлен workflow `.github/workflows/android-release.yml`.
- В CI используется Temurin JDK 17.
- В CI используется Gradle cache через `gradle/actions/setup-gradle`.
- Для загрузки workflow artifact используется `actions/upload-artifact@v4`.
- Для публикации APK в GitHub Releases используется `softprops/action-gh-release`.

### Проверено

- Локальная сборка проходит через `assembleDebug`.
- Unit-тесты проходят через `testDebugUnitTest`.
- GitHub Actions успешно собирает APK.
- GitHub Release содержит APK-файл для скачивания.

## 0.3.0

### Добавлено

- Добавлена история операций.
- Добавлено сохранение последних конвертаций.
- Добавлено отображение режима, направления, алфавита, смещения, ввода и результата в истории.
- Добавлена возможность повторить операцию из истории.
- Добавлена возможность скопировать результат из истории.
- Добавлена возможность очистить историю.

### Улучшено

- История ограничена последними 20 операциями.
- Пустой ввод и пустой результат не добавляются в историю.
- Подряд идущие полные дубли операций не добавляются повторно.
- Повтор старой операции после других действий сохраняется как отдельная запись.

### Проверено

- Приложение успешно собирается через `assembleDebug`.
- Unit-тесты проходят через `testDebugUnitTest`.
- Приложение запускается на Android Emulator.
- Проверено сохранение, повтор, копирование и очистка истории.
- Проверено, что основной конвертер и справка продолжают работать.
